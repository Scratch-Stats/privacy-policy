Privacy Policy

Last updated: 25 June 2026

Scratch Stats (“we”, “our”, “the site”) is a community‑made website that displays public information from the Scratch platform. This Privacy Policy explains what information we collect, how we use it, and how we keep it safe.

1. Information We Collect

1.1 Public Scratch Data

When you search for a Scratch user, project, or studio, we request information from the public Scratch API. This includes:

  • Usernames
  • Project titles and statistics
  • Studio information
  • Public profile details

This data is already publicly visible on Scratch.

1.2 Website Usage Data

We may collect basic, non‑personal information such as browser type, device type, and general usage statistics. This helps us improve the site.

1.3 No Personal Accounts

Scratch Stats does not create user accounts, store passwords, or require sign‑in for regular visitors.

2. How We Use Information

We use collected information to:

  • Display Scratch statistics
  • Improve site performance
  • Fix bugs and prevent abuse

We do not sell, trade, or share your data with advertisers.

3. Cookies & Local Storage

Scratch Stats may use LocalStorage or SessionStorage to save admin settings or temporary sessions. These are stored only on your device and never sent to our servers.

We do not use tracking cookies or advertising cookies.

4. Third‑Party Services

4.1 Scratch API

We fetch public data from the Scratch API at https://api.scratch.mit.edu/. We do not store or modify private data.

4.2 GitHub Pages

The site is hosted on GitHub Pages, which may collect basic technical logs for security and performance.

5. Data Security

We take reasonable steps to keep the site safe. We do not collect sensitive personal information, and we do not store visitor data on a server.

6. Children’s Privacy

Scratch Stats is designed for the Scratch community, including children and teens. We do not collect personal information, use targeted advertising, or require accounts.

7. Changes to This Policy

We may update this Privacy Policy from time to time. The “Last updated” date will always reflect the latest version.
